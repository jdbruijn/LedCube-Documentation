Evaluate an assertion.

If the argument expression of this macro with functional form compares equal to zero (i.e., the expression is false), a message is written to the standard device and the program execution is halted. Halting the program execution is done by setting the CPU interrupt priority level to seven and entering an infinite loop.

Note
This macro is disabled if, at the moment of including "MyAssert.h", a macro with the name NDEBUG has already been defined. This allows for a coder to include as many assert calls as needed in a source code while debugging the program and then disable all of them for the production version by simply including a line like #define NDEBUG at the beginning of the code, before the inclusion of "MyAssert.h". Therefore, this macro is designed to capture programming errors, not user or run-time errors, since it is generally disabled after a program exits its debugging phase.
